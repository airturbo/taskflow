## 1. 应用启动流程

1. 进入应用
2. 默认落到“今日”或上次停留的工作区
3. 左侧立即可见清单、标签、过滤器、工具
4. 中央展示当前视图
5. 右侧自动保留任务详情/专注工具位

成功标准：
- 首屏就知道自己现在在哪个工作区
- 无需跳页即可开始创建任务

---

## 2. 快速创建流程

1. 用户在顶部快速输入框输入：如“明天下午 3 点产品评审”
2. 系统解析出标题与日期时间
3. 用户可补充优先级 / 清单 / 标签
4. 按 Enter 立即创建
5. 新任务自动出现在当前视图，并在右侧详情面板选中

降级策略：
- 解析失败时保留原文为标题，不阻塞创建

---

## 3. 任务详情编辑流程

1. 点击任意任务
2. 右侧面板展开
3. 在同一面板中完成：
   - 标题与描述编辑
   - 子任务维护
   - 标签 / 清单 / 优先级修改
   - 开始/截止、重复、提醒设置
   - 预计番茄与附件占位维护
4. 修改自动保存到本地
5. 主视图即时同步刷新

---

## 4. 视图切换流程

### 4.1 列表 → 日历
1. 顶部视图控件切到日历
2. 保留当前筛选上下文
3. 同一批任务以月/周/日程形式展示

### 4.2 列表 → 看板
1. 切换到看板
2. 任务按状态分列
3. 用户修改状态或迁移列后，同步写回统一任务源

### 4.3 列表 → 时间线 / 四象限
1. 切换视图
2. 任务按时间跨度或紧急/重要规则重新映射
3. 视图变化不改变任务实体本身，只改变展示逻辑

---

## 5. 过滤与聚焦流程

1. 在侧边栏点击标签、清单或智能清单
2. 中央视图切换上下文
3. 顶部搜索继续叠加关键词过滤
4. 用户可在不同视图间保留上下文继续工作

目标：
- 让“筛选”成为工作方式，而不是一个埋得很深的功能点

---

## 6. 番茄专注流程

1. 用户在右侧或专注页选择一个任务
2. 设置/查看预计番茄数
3. 点击开始
4. 计时结束后：
   - 当前番茄数 +1
   - 任务累计专注分钟数
   - 切换到休息态
5. 统计页同步更新

---

## 7. 习惯打卡流程

1. 用户进入习惯页
2. 看到今天的全部习惯卡片
3. 根据不同类型进行：
   - 是/否打卡
   - 次数累加
   - 时长累加
4. 页面实时反馈连续天数与今日进度
5. 统计区同步反映本日达成情况

---

## 8. 协作形态流程（本地化 Demo）

1. 打开任务详情
2. 可查看 assignee、评论、活动流字段
3. 用户可追加一条评论或变更 assignee
4. 当前 Demo 中只做本地状态反馈，不做多人实时分发

---

## 9. 验收路径

建议按以下路径做最终验收：
1. 快速创建一条带日期的任务
2. 打开详情补子任务、提醒和预计番茄
3. 切换到看板并改变状态
4. 切换到时间线查看任务跨度
5. 切换到四象限查看自动分类
6. 开始一次番茄专注
7. 完成一项习惯打卡
8. 打开统计页确认数据变化

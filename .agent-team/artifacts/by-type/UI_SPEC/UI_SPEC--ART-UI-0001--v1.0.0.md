## 1. 设计原则

| # | 原则 | 说明 |
|---|------|------|
| D1 | **效率优先** | 减少操作步骤——快速录入 ≤ 3 秒、一键完成、拖拽排序 |
| D2 | **信息密度平衡** | 列表紧凑但不拥挤；功能丰富但不嘈杂；默认简洁、按需展开 |
| D3 | **视觉层级清晰** | 颜色、字重、间距建立明确层级——优先级一眼可辨 |
| D4 | **一致性** | 全应用统一交互模式、组件样式、动效节奏 |
| D5 | **温暖专业** | 偏向 Linear/Notion 的克制美学，"可信赖的伙伴"感 |
| D6 | **渐进披露** | 基础功能即时可用，高级功能按需发现 |

| `--duration-normal` | 200ms | 位移/大小变化 |
| `--duration-slow` | 350ms | 面板滑入/页面切换 |

### 5.2 核心组件详细规范

#### TaskRow 任务行

```
Props:
  - task: Task                 // 任务数据
  - mode: "compact" | "comfortable"  // 紧凑40px / 舒适52px
  - selected: boolean          // 是否被选中
  - dragging: boolean          // 是否拖拽中

States:
  - default     → 正常展示
  - hover       → 背景 brand-50, 显示拖拽手柄+操作按钮
  - selected    → 背景 brand-100, 左边框 2px brand-500
  - dragging    → shadow-xl, 半透明, 旋转 2deg
  - completed   → 文字灰色+划线, 复选框打勾动画

交互:
  - 单击       → 展开任务详情面板
  - 复选框点击  → 切换完成状态
  - 右键       → 弹出 ContextMenu
  - 长按/拖拽   → 进入拖拽排序模式
  - Ctrl+点击   → 多选
```

#### Button 按钮

```
尺寸:
  - sm: H 28px, px 12px, 13px text
  - md: H 36px, px 16px, 14px text
  - lg: H 44px, px 20px, 16px text

变体:
  - primary:    bg brand-500, text white, hover brand-600
  - secondary:  bg gray-100, text gray-700, hover gray-200, border gray-200
  - ghost:      bg transparent, text gray-700, hover gray-100
  - danger:     bg danger, text white, hover danger-dark
  - icon-only:  圆形, 仅图标

通用规则:
  - 圆角 radius-md (8px)
  - 点击有 scale(0.97) 反馈
  - 禁用态: opacity 0.5, cursor not-allowed
  - loading态: 显示 spinner, 文字保留
```

## 8. 无障碍设计

| 规范 | 要求 |
|------|------|
| 对比度 | 所有文本与背景对比度 ≥ 4.5:1 (WCAG AA) |
| 键盘导航 | 所有交互元素可通过 Tab 到达，Enter/Space 触发 |
| Focus Ring | 所有可聚焦元素有可见的 2px brand-500 focus ring |
| ARIA | 语义化标签 + aria-label / aria-describedby |
| Screen Reader | 任务状态变化有 aria-live 通知 |
| 动效减弱 | 遵循 `prefers-reduced-motion`，减少动画 |

| 拖拽开始 | 浮起 + 阴影扩大 + 旋转2° | 150ms | ease-out |
| 拖拽释放 | 弹入目标位置 | 250ms | ease-spring |

| 页面加载 | 骨架屏 → 内容淡入 | 300ms | ease-out |
| 按钮点击 | scale(0.97) → scale(1) | 100ms | ease-spring |

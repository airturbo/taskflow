## 1. 设计原则

### 1.1 视觉立场
不做“轻飘飘的 SaaS 模板”，而做一个像真正每天会打开数十次的生产力控制台：
- 深色底 + 高可读卡片层级
- 左导航明确、中央内容高效、右侧上下文持续在线
- 强调信息密度，但用色块、留白、字号层级保证不乱

### 1.2 一眼识别点
用户打开页面后，第一眼应该记住三件事：
1. 这不是简单待办，而是完整工作台
2. 任务、视图、工具在一个界面里无缝切换
3. 高级能力默认就在手边，没有会员门槛

### 1.3 交互优先级
1. 快速录入
2. 快速筛选
3. 快速切换视图
4. 快速编辑任务详情
5. 快速进入专注 / 习惯 / 统计

## 2. 设计系统

### 2.1 色彩
```css
--bg: #0b1020;
--panel: rgba(17, 24, 39, 0.88);
--panel-soft: rgba(30, 41, 59, 0.72);
--line: rgba(148, 163, 184, 0.18);
--text: #e5eefc;
--muted: #92a3bf;
--accent: #7c9cff;
--accent-2: #54d2a0;
--danger: #ff6b7a;
--warning: #ffb454;
--violet: #a78bfa;
```

### 2.2 字体策略
- 标题：`Iowan Old Style`, `Palatino Linotype`, serif
- 正文：`Avenir Next`, `PingFang SC`, `Helvetica Neue`, sans-serif
- 数字：等宽变体，用于番茄计时与统计数字

### 2.3 组件调性
- 圆角：16px 主卡片，12px 次级卡片，10px 输入控件
- 阴影：柔和深色扩散，不做悬浮塑料感
- 边框：低对比度半透明边框，用于界面分层

### 3.1 Sidebar
包含四段：
1. 系统视图（今日、最近 7 天、收件箱、已完成、回收站）
2. 文件夹与清单
3. 标签与智能清单
4. 工具入口（番茄、习惯、统计）

### 3.2 Top Bar
包含：
- 全局搜索
- 快速创建输入框
- 视图 segmented control
- 当前上下文标题
- 主题切换 / 命令入口

### 3.3 Right Rail
上半区：番茄与专注摘要
下半区：任务详情面板、子任务、评论、活动流

## 4. 核心页面

### 4.1 列表视图
- 采用可分组的任务列表
- 支持 `Overdue / Today / Upcoming` 分组展示
- 每行显示：复选框、优先级旗、标题、标签、日期、子任务进度、所属清单、预计番茄
- hover 时显示更多操作

### 4.2 日历视图
- 提供 `月 / 周 / 日程列表` 三个层级
- 月视图重在任务分布与日期聚合
- 周视图重在时间块与负荷感知
- 日程列表重在按日展开任务
- 当前 Demo 允许使用轻量交互表达代替完整拖拽

### 4.3 看板视图
- 默认三列：待办 / 进行中 / 已完成
- 卡片显示：标题、优先级、日期、标签、子任务进度、预计番茄
- 支持列间迁移

### 4.4 时间线视图
- 横轴按日期刻度展示
- 行级显示任务跨度条
- 起止时间不存在时，用单点节点代替跨度条

### 4.5 四象限视图
- 2x2 宫格布局
- 根据优先级 + 截止时间自动分类
- 每个象限包含行动提示：立即处理 / 规划安排 / 委派 / 放弃

### 4.6 习惯页面
- 顶部显示今日完成率
- 卡片显示：图标、目标、进度、连续天数
- 支持布尔 / 计数 / 时长三种模式

### 4.7 统计页面
- 指标卡：活跃任务、完成率、专注时长、习惯完成率
- 趋势图：任务完成趋势、专注分布、习惯热力表达
- 使用轻量图形表达，不引入重图表库

## 5. 核心组件规范

### 5.1 快速创建条
- 常驻在顶部
- 输入支持自然语言提示
- 右侧有属性快捷 chips：清单 / 优先级 / 日期
- Enter 创建，Shift+Enter 保留输入

### 5.2 任务卡
- 默认高密度紧凑样式
- 标题 15px、描述 13px
- 优先级以左边缘色线和旗标同时表达
- 完成态使用低对比度划线，不做夸张删除感

### 5.3 详情面板
必须包含：
- 标题
- 描述/笔记
- 子任务
- 开始/截止
- 重复
- 提醒
- 标签
- 清单
- 预计番茄
- 附件占位
- 评论与动态

### 5.4 番茄面板
- 大号计时数字
- 关联任务显示预计/已完成番茄
- 白噪音入口做成 pill 开关
- 当前状态明确：专注 / 休息

---

## 6. 响应式策略
- ≥ 1440px：标准三栏
- 1024px - 1439px：右栏可折叠
- < 1024px：切换为双栏或堆叠视图，详情面板改抽屉

---

## 7. 可用性约束
- 主要流程必须可通过键盘完成
- 日期、优先级、完成态需具备明显视觉区分
- 空状态不应“空白”，而应给出下一步引导
- 所有高级能力默认可见，不用埋进深层设置

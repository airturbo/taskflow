## 1. 项目目标

在当前 PRD v2.0 与 UI_SPEC v1.0 的基础上，将项目推进到**可用状态**。这里的“可用”定义为：

> 用户可以在浏览器中直接使用一个**单人本地优先 MVP**，完成任务录入、清单管理、今日视图、搜索筛选、基础看板、番茄专注等核心流程，并具备稳定的数据持久化与基本可用性验证。

这不是最终全功能版，而是**第一版可用产品（usable MVP）**。

## 4. 交付定义

### 可用版完成标准
1. 本地 `npm install && npm run dev` 可启动。
2. 能新增、编辑、完成、删除、检索任务。
3. 能按清单查看任务，并有“今日”聚合。
4. 数据刷新后不丢失。
5. 至少存在一个可切换的聚合视图（列表 / 看板）。
6. 番茄计时器可运行。
7. `npm run build` 成功。

## 5. 风险与决策

| 风险 | 决策 |
|------|------|
| PRD 范围过大 | 以“单人可用 MVP”切分首发范围，剩余能力保留扩展位 |
| 多角色治理数据不一致 | 先修复状态与审计，再推进后续阶段 |
| 协作/同步开发成本高 | 首发不做账号体系，优先本地优先体验 |
| 日历/拖拽复杂度高 | 首发只实现简化看板，不做完整日历拖拽 |

## 1. 背景与目标

### 1.1 项目背景

滴答清单（TickTick）是当前市场上功能最全面的待办管理工具之一，集成了任务管理、日历视图、番茄专注、习惯打卡、四象限、看板、时间线等完整功能体系。但其高级功能（日历、过滤器、扩展容量、协作、统计等）需要付费订阅（¥139/年）。

**我们的机会**：打造一款**对标滴答清单全功能体系**的待办管理工具，将滴答清单的免费功能 + 付费高级功能**全部免费提供**，形成极强的竞争优势。

### 1.2 产品愿景

> **一句话定位**：滴答清单的全功能平替——一切高级功能，全部免费。

### 1.3 核心目标

| # | 目标 | 衡量指标 |
|---|------|---------|
| G1 | 功能对标滴答清单 | 覆盖滴答清单全部核心功能（含付费功能） |
| G2 | 极速任务录入 | 新建一条待办 ≤ 3 秒 |
| G3 | 多视图任务管理 | 支持列表/日历/看板/时间线/四象限 5 种视图 |
| G4 | 专注与习惯一体化 | 内置番茄专注 + 习惯打卡 |
| G5 | 完全免费无限制 | 无清单数量、任务数量、协作人数等限制 |

## 3. 功能模块详细设计

### 3.1 📝 任务管理（核心）

#### 3.1.1 任务创建

| 功能点 | 详细说明 |
|--------|---------|
| 快速创建 | 点击 "+" 或快捷键（Ctrl/Cmd + N），弹出创建面板，光标自动聚焦 |
| 标题智能识别 | 输入"明天下午3点开会"自动识别日期时间并设置提醒 |
| 可选属性 | 清单归属、优先级、截止日期、标签、描述 |
| 回车创建 | 按回车直接创建，无需鼠标操作 |
| 连续创建 | 创建后输入框不关闭，支持连续添加多条 |
| 默认收件箱 | 未指定清单的任务自动归入"收件箱" |

#### 3.1.2 任务属性

| 属性 | 类型 | 说明 |
|------|------|------|
| 标题 | 文本（必填） | 支持智能日期识别 |
| 描述/笔记 | 富文本 | 支持 Markdown 格式 |
| 清单 | 选择 | 归属哪个清单 |
| 优先级 | 枚举 | 🔴 紧急 / 🟡 高 / 🔵 普通 / ⚪ 低 |
| 标签 | 多选 | 可添加多个彩色标签 |
| 截止日期 | 日期时间 | 支持仅日期或日期+时间 |
| 开始日期 | 日期时间 | 可选，用于时间线和日历视图 |
| 提醒 | 多个时间 | 支持设置多个提醒时间点 |
| 重复规则 | 周期 | 每天/每周/每月/每年/自定义/工作日 |
| 子任务 | checklist | 支持无限子任务，子任务也可设提醒 |
| 附件 | 文件 | 支持图片、文档等附件上传，不限数量 |
| 指派人 | 用户 | 协作模式下可指派给成员 |

#### 3.1.3 任务操作

| 操作 | 说明 |
|------|------|
| 完成/取消完成 | 点击复选框切换，完成时有动画反馈 |
| 编辑 | 点击展开详情面板，所有修改自动保存 |
| 删除 | 软删除至回收站，支持撤销（底部 toast） |
| 拖拽排序 | 拖拽调整顺序，有占位符视觉反馈 |
| 拖拽移动 | 拖拽到不同清单、日历日期 |
| 批量操作 | 多选后批量删除/移动/设标签/设优先级 |
| 复制任务 | 复制为新任务 |

### 3.2 📂 组织系统

#### 3.2.1 清单

#### 3.2.2 标签

|--------|------|

| 标签筛选 | 侧边栏点击标签查看该标签下的所有任务 |
| 标签管理 | 编辑/删除/合并标签 |

#### 3.2.3 过滤器（对标滴答付费功能）

| 功能点 | 说明 |
|--------|------|

#### 3.2.4 排序与分组

| 功能点 | 说明 |
|--------|------|

## 6. 技术方向建议

| 维度 | 建议 | 理由 |
|------|------|------|
| **平台** | Web（PWA）优先 | 一套代码覆盖桌面+移动端，支持离线 |
| **前端框架** | React + TypeScript | 组件化、类型安全、生态丰富 |
| **状态管理** | Zustand 或 Jotai | 轻量、性能好、适合复杂视图 |
| **UI 框架** | Tailwind CSS + Radix UI | 高度可定制、无障碍支持 |
| **数据库** | IndexedDB (Dexie.js) | 本地优先，大容量离线存储 |
| **云同步** | Supabase / 自建 | 实时同步（WebSocket）、用户认证 |
| **日历** | 自研 or FullCalendar | 满足多视图和拖拽需求 |
| **看板** | dnd-kit | React 拖拽库，性能优秀 |
| **富文本** | TipTap (Markdown) | 支持 Markdown，可扩展 |
| **音频** | Howler.js | 白噪音播放和混合 |

## 10. 风险与缓解

| # | 风险项 | 影响 | 缓解策略 |
|---|--------|------|---------|
| R1 | 功能范围大，开发周期长 | 交付延期 | 严格按里程碑分期交付，P0 先行 |
| R2 | IndexedDB 存储限制 | 大量数据时性能下降 | 引入虚拟滚动、分页加载、定期归档 |
| R3 | 多视图复杂度高 | 状态同步困难 | 单一数据源 + 派生视图模式 |
| R4 | 免费模式无收入 | 长期运营成本 | 后期引入可选捐赠/企业版 |
| R5 | 实时协作冲突 | 数据一致性问题 | 采用 CRDT 或 OT 算法处理冲突 |

## 背景

本文件把 4 轮 UX 迭代中确认的交互决策从"隐含在代码修复中"升级为正式产品规则，供设计、前端、QA 和后续迭代对齐。

## 规则清单

### 规则 1：创建后保持当前工作区

- **场景**：在"今日""最近 7 天"等情境视图中创建任务。
- **规则**：创建成功后，用户仍停留在当前工作区。不强制跳转到任务所属清单。
- **反馈**：系统给出明确的创建成功反馈，说明任务已进入哪个清单，以及新任务当前是否出现在工作区结果中。
- **验收标准**：
  1. 在"今日"创建任务后，topbar 标题仍为"今日"。
  2. 如果新任务在当前视图中可见，显示绿色正向反馈。
  3. 如果新任务不在当前视图中可见，显示黄色提示并提供"查看任务"入口。
  4. 反馈条在 5 秒内自动消退。

### 规则 2：行动视图默认只展示未完成项

- **场景**：今日、最近 7 天、收件箱、普通清单、标签视图、智能清单。
- **规则**：以上"行动视图"默认只显示 `completed === false` 的任务。
- **已完成项**：进入独立的"已完成"系统视图，不混入行动视图。
- **计数一致性**：左侧导航计数与主区显示的任务数必须使用同一套过滤规则，不允许各说各话。
- **验收标准**：
  1. 在看板或四象限中将任务改为"已完成"后，该任务退出当前行动视图。
  2. 左侧计数同步减少。
  3. 进入"已完成"视图可以看到该任务。

### 规则 3：恢复回到最合理的工作区

- **场景**：从回收站恢复任务。
- **规则**：恢复后，系统根据任务当前状态选择落点——
  - 已完成任务 → 回到"已完成"视图
  - 未完成且今日到期/逾期 → 回到"今日"
  - 未完成且 7 天内到期 → 回到"最近 7 天"
  - 其他 → 回到任务所属清单
- **验收标准**：
  1. 恢复后用户立即看到任务出现在工作区中。
  2. 不会出现"恢复成功但任务消失"的体验。

### 规则 4：番茄专注默认跟随当前详情任务

- **场景**：进入番茄专注工具。
- **规则**：
  - 未开始时，番茄关联任务默认为当前右栏详情选中的任务（`selectedTaskId`）。
  - 正在计时时，任务锁定不变。
  - 如果用户切换了详情任务但番茄仍在计时，给出同步提示。
- **验收标准**：
  1. 从任意任务详情点"开始专注"后，番茄主任务与该任务一致。
  2. 右栏和主区关联的任务不互相打架。

## 后续使用

本文件作为 `PRD` 的补丁存在，后续正式版本的 PRD 应将这些规则纳入交互章节。设计师、前端和 QA 的交付物均应引用本规则。

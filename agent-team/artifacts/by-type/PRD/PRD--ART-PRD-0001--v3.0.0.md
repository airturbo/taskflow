## 1. 背景与决策

上一轮交付仅完成了“单人本地优先 usable MVP”，能跑，但与“滴答清单平替”目标存在明显落差：视图覆盖不足、任务引擎过薄、组织系统弱、专注/习惯/统计只停留在点状能力，整体体验更像 demo 骨架，不像可用产品。

本轮明确做出三个决策：

1. **目标回正**：产品目标不再是“最小可用”，而是**对照滴答清单核心功能体系做高保真复刻**。
2. **商业化去除**：不设置会员墙，不区分免费/付费能力；滴答清单 Premium 能力在本产品中默认开放。
3. **交付方式分层**：
   - **产品目标**：功能对齐滴答清单；
   - **本轮交付**：先完成一个**高保真可交互 Web Demo**，覆盖核心任务、组织、多视图、专注、习惯、统计与高级属性链路；
   - **后续补齐**：外部集成、真实多端同步、多人实时协作、原生平台特性。

一句话定位：

> **滴答清单功能平替版，去掉商业化门槛，把高级能力全部作为默认能力开放。**

## 3. 对齐原则

### 3.1 直接对齐的能力
- 任务快速录入、任务详情编辑、子任务、标签、截止/开始时间、重复、提醒、附件占位
- 收件箱 / 今日 / 最近 7 天 / 已完成 / 回收站
- 文件夹 + 清单 + 标签 + 过滤器 + 排序/分组
- 列表 / 日历 / 看板 / 时间线 / 四象限
- 番茄专注、预计番茄数、白噪音入口、专注累计
- 习惯打卡、连续天数、热力/进度表达
- 统计面板、任务趋势、专注统计、习惯统计
- 快捷创建、搜索、命令式操作入口

### 3.2 默认免费的高级能力
- 完整多视图日历
- 自定义过滤器 / 智能清单
- 子任务提醒
- 高级统计
- 主题与个性化
- 预计番茄数与高级专注面板

### 3.3 平台限制下的处理原则
以下能力作为**产品目标保留**，但在本轮 Web Demo 中采取替代或降级策略：
- **位置提醒**：保留数据模型与入口，不做真实地理围栏
- **锁屏悬浮球/小组件**：以 Web 快捷入口与固定工具位替代
- **真实多人实时协作**：先完成本地协作字段、评论、活动流与指派模型
- **外部日历双向同步**：先完成订阅入口与占位流程，不接第三方 API

## 4. 功能模块定义

### 4.1 任务引擎
**目标**：让任务本身具备接近滴答清单的完整属性体系，而不是只剩标题和日期。

**必须具备：**
- 快速创建 + 回车创建 + 连续创建
- 标题中的日期词解析（至少支持今天/明天/后天/几点）
- 描述/笔记
- 清单、标签、优先级、开始时间、截止时间
- 重复规则
- 多提醒
- 子任务
- 附件占位
- 指派、评论、活动流占位
- 软删除、恢复、完成、复制、拖拽迁移（Demo 允许以显式操作替代部分拖拽）

### 4.2 组织系统
**必须具备：**
- 文件夹 + 清单 + 系统清单
- 标签管理
- 保存过滤器为智能清单
- 按优先级 / 日期 / 标题排序
- 按清单 / 标签 / 状态分组
- 每个视图记忆自己的筛选条件

### 4.3 多视图工作区
**必须具备：**
- 列表视图：日常管理主视图
- 日历视图：月 / 周 / 日程列表至少三种展示层次
- 看板视图：按状态分列，支持任务迁移
- 时间线视图：按开始/结束时间表达任务跨度
- 四象限视图：按紧急/重要自动归类，并允许手动调整

### 4.4 专注、习惯与统计
**必须具备：**
- 番茄计时器：开始 / 暂停 / 重置 / 跳过 / 关联任务
- 预计番茄数、已完成番茄数、白噪音入口
- 习惯：布尔 / 计数 / 时长三种记录方式
- 统计：任务完成、专注、习惯三类指标

### 4.5 协作与同步
**产品目标：**
- 清单分享、任务指派、评论、动态、多人协作
- 数据导入导出、备份恢复、云同步、离线能力

**Demo 策略：**
- 先完整提供字段、交互入口和本地假数据反馈
- 不在本轮接入真实账户体系和实时同步

## 6. 本轮 Demo 验收标准

### 6.1 必须通过
1. 启动后直接进入完整三栏工作台，而不是模板页。
2. 可以快速创建任务，并能解析基础中文日期词。
3. 任务详情可编辑：标题、描述、清单、标签、优先级、开始/截止、重复、提醒、子任务、预计番茄数。
4. 至少支持列表 / 日历 / 看板 / 时间线 / 四象限五个视图中的四个真实可交互展示。
5. 支持习惯打卡、番茄计时和统计看板。
6. 支持过滤器、标签查看、系统清单切换。
7. 所有数据本地持久化，刷新不丢失。
8. `npm run build` 成功。

### 6.2 可接受的 Demo 限制
- 不接第三方账号与真实多人协作
- 不做真实浏览器推送通知与位置提醒
- 附件先以文件名占位，不做真实上传存储
- 外部日历 / 导入导出先保留入口

## 8. 风险与策略

| 风险 | 说明 | 应对 |
|------|------|------|
| 范围极大 | 一次性追平生产级能力不可行 | 以“高保真可交互 Demo”作为本轮交付线 |
| 视图复杂 | 日历/时间线/四象限实现成本高 | 统一单一数据模型，多视图派生 |
| 协作能力难以真做 | 实时同步依赖后端 | 先完成本地化协作形态与字段 |
| Web 平台限制 | 位置提醒、小组件、原生通知存在限制 | 明确替代方案，不伪造能力 |

## 变更说明

本版本在 v2.0.0 基础上新增以下设计说明，源自 4 轮 UX 迭代中设计师参与的稀疏态 polish 和创建反馈策略。

## 1. 多视图稀疏态设计

### 1.1 ViewSummary 摘要条

所有非列表视图（日历、时间线、四象限）在顶部增加统一的"视图摘要条"：

- **eyebrow**：视图类型标识（如 `calendar · week`、`timeline`、`matrix`）
- **标题**：当前视图的关键视角说明
- **描述**：1-2 句话解释当前视图在低密度场景下仍然有用的原因
- **指标卡**：3 个小指标（任务数、落点/覆盖日期、最近下一项），使用 `summary-metric` 组件

视觉规范：
- 容器：`border-radius: 20px`，`background: var(--panel-soft)`，`padding: 16px`
- 指标卡：三列 grid，每卡 `border-radius: 16px`，字号 span 12px / strong 20px

### 1.2 SparseGuide 引导面板

当视图内任务数 ≤ 3 时，自动显示"稀疏引导"面板：

- **eyebrow**：`sparse state polish`
- **标题 + 描述**：解释当前视图负荷偏轻时的价值
- **标签组**：3 个 `mini-tag` 提示用户可以做什么（看空档、保留节奏感、安排下一步）

### 1.3 日历空单元

月视图中无任务的日期单元格显示一句淡灰文案：
> 暂无安排，可留给缓冲或深度工作。

周视图中无任务的列显示：
> 今天没有硬性安排，可以预留给缓冲、回顾或深度工作。

### 1.4 四象限空区域

空象限不再是纯留白，而显示 `matrix-placeholder` 组件：
- **标题**：当前为空
- **说明**：针对每个象限给出不同解释（如"这里应该只留真正需要马上动手的任务"）

## 2. 快速创建反馈条

### 2.1 组件：`quick-feedback`

创建任务后，在 composer-bar 下方显示反馈条（4.8 秒后自动消退）：

- **正向状态**（`is-positive`）：新任务在当前工作区可见
  - 背景：`linear-gradient(135deg, rgba(84, 210, 160, 0.12), rgba(124, 156, 255, 0.1))`
  - 文案：`仍停留在"今日"，并已把新任务放进当前工作流。`

- **提示状态**（`is-warning`）：新任务不在当前工作区
  - 背景：`linear-gradient(135deg, rgba(255, 180, 84, 0.12), rgba(124, 156, 255, 0.08))`
  - 文案：`任务已进入"收件箱"，你仍停留在"今日"。这条任务暂时不在当前结果里。`
  - 操作按钮：`查看任务`（跳到任务所在清单）+ `知道了`（关闭反馈）

---

## 3. 番茄同步提示

### 3.1 组件：`focus-sync-tip`

当番茄未在计时，且历史关联任务与当前详情任务不一致时显示：

- 容器：`border-radius: 16px`，`background: rgba(124, 156, 255, 0.08)`
- 文案：`右栏已切到新任务，下一次开始会自动跟随当前详情。`
- 操作按钮：`立即同步`

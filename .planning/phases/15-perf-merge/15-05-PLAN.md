# Plan 15-05: Field-level Merge Engine

## Objective
实现 field-level merge 引擎：逐字段比较，单方修改自动合并，双方同字段冲突收集供 UI 处理。

## Tasks

<task id="15-05-A">
### A: Create utils/field-merge.ts
- Define `MERGEABLE_FIELDS`: all scalar Task fields (title, note, status, priority, dueAt, startAt, deadlineAt, completed, listId, assignee, repeatRule, estimatedPomodoros)
- Define `MergeResult` type:
  ```typescript
  interface MergeResult {
    merged: Task
    conflicts: FieldConflict[]
  }
  interface FieldConflict {
    taskId: string
    field: string
    localValue: unknown
    remoteValue: unknown
    localUpdatedAt: string
    remoteUpdatedAt: string
  }
  ```
- Implement `mergeTask(local: Task, remote: Task): MergeResult`
  - For each field: compare fieldVersions timestamps
  - If local.fieldVersions[f] > remote.fieldVersions[f]: local wins (no conflict)
  - If remote.fieldVersions[f] > local.fieldVersions[f]: remote wins (no conflict)  
  - If equal or missing: compare values; if same → no conflict; if different → conflict
  - Merged task gets winner values + updated fieldVersions
- Implement `mergeTaskList(localTasks: Task[], remoteTasks: Task[]): { tasks: Task[]; conflicts: FieldConflict[] }`
  - Handle new tasks (exists only in remote: add)
  - Handle deleted tasks (exists only locally: keep — deletion handled separately)
  - Handle common tasks: run mergeTask
</task>

<task id="15-05-B">
### B: Integrate merge in App.tsx handleRemoteUpdate
- Replace current whole-task updatedAt comparison with mergeTaskList
- Collect all conflicts
- If conflicts: trigger conflict resolution UI (store in state)
- Apply non-conflicting merged tasks immediately
</task>

<task id="15-05-C">
### C: Update useTaskActions or App.tsx to stamp fieldVersions on mutation
- When a task field is updated: set `fieldVersions[field] = new Date().toISOString()` for each changed field
- Helper: `stampFieldVersions(task, patch): Task` in field-merge.ts
</task>

# Plan 15-06: Conflict Resolution UI

## Objective
冲突解决 UI：双方同字段冲突并排展示，用户选择每个字段的胜出值。

## Tasks

<task id="15-06-A">
### A: Create components/ConflictResolutionDialog.tsx
- Props: `conflicts: FieldConflict[]`, `onResolve: (resolutions: Record<string, 'local' | 'remote'>) => void`, `onDismiss: () => void`
- Group conflicts by taskId
- For each task: show task title, then list conflicting fields side by side
- Each field row: field name | local value | remote value | radio/button to choose
- Footer: "应用选择" button (disabled until all fields resolved) + "忽略" button
- Use existing dialog CSS patterns (modal overlay)
</task>

<task id="15-06-B">
### B: Add ConflictResolutionDialog state to App.tsx / WorkspaceApp
- State: `pendingConflicts: FieldConflict[] | null`
- Show dialog when pendingConflicts !== null
- onResolve: apply chosen values to tasks, clear conflicts
- onDismiss: discard remote values for conflicting fields, clear conflicts
</task>

<task id="15-06-C">
### C: Create ConflictResolutionDialog.module.css
- Styles for dialog, side-by-side comparison layout, conflict row, winner selection
</task>

<task id="15-06-D">
### D: Commit
- Message: `feat(sync): conflict resolution UI for field-level merge conflicts`
</task>

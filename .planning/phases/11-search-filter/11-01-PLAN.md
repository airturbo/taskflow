# Plan 11-01: 扩展 FilterState 模型

## Objective
扩展 useFilterState hook，加入 priority/status/due 过滤维度，
并将新字段接入 useWorkspaceComputed 过滤逻辑。

## Tasks

<tasks>
  <task id="11-01-A">
    扩展 useFilterState：加入 filterPriority, filterStatus, filterDue 字段
    及对应的 setter/clear 方法
  </task>
  <task id="11-01-B">
    扩展 useWorkspaceComputed 接收并应用新过滤字段
    （matchesPriority, matchesStatus, matchesDue）
  </task>
  <task id="11-01-C">
    在 App.tsx 中将新字段透传到 useWorkspaceComputed
  </task>
</tasks>

## Implementation Notes
- Priority 用 Priority[] (空数组=不过滤)
- Status 用 TaskStatus[] (空数组=不过滤)
- Due 用 'overdue'|'today'|'week'|'none'|null (null=不过滤)
- matchesDue 需要 getDateKey() 比较

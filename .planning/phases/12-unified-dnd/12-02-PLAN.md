# Plan 12-02: DndContext 提升到 WorkspaceShell

## 目标

将 KanbanView 内嵌的 DndContext 提升到 WorkspaceShell，实现共享 DndProvider。
KanbanView 变为纯 presentational，不再持有 DndContext。

## Tasks

<task id="12-02-T1">
  <title>WorkspaceShell — 添加 DndContext + activeKanbanTaskId + DragOverlay</title>
  <read_first>
    - web/src/components/WorkspaceShell.tsx
    - web/src/components/WorkspaceViewContent.tsx
    - web/src/components/views/KanbanView.tsx
  </read_first>
  <action>
    1. 在 WorkspaceShell.tsx 中：
       - import DndContext, DragOverlay, MouseSensor, TouchSensor, useSensors, useSensor from @dnd-kit/core
       - import useState
       - 在 WorkspaceShell 函数体内添加：
         - activeKanbanTaskId: string | null = useState(null)
         - sensors = useSensors(MouseSensor { distance: 6 }, TouchSensor { delay: 200, tolerance: 5 })
         - handleKanbanDragEnd: 从 active.id + over.id 调用 p.applyKanbanDropFeedback
       - 用 DndContext 包裹整个渲染树（最外层 div 外层，或 return 的根元素外层）
       - 在渲染树末尾添加 DragOverlay（渲染 activeKanbanTask 的卡片预览）

    2. KanbanView.tsx 中：
       - 移除内置 DndContext、useSensors、sensors、handleDragEnd、activeTaskId state
       - 移除 DragOverlay（提升到 WorkspaceShell）
       - KanbanView 接收新 prop: activeTaskId: string | null（由 WorkspaceShell 传入）
       - KanbanDraggableCard 保留 useDraggable
       - KanbanDroppableColumn 保留 useDroppable
       - KanbanView 整体只渲染 kanbanGrid div（不再包裹 DndContext）

    3. WorkspaceViewContent.tsx 向 KanbanView 传递 activeTaskId prop

    4. WorkspaceShell.tsx 向 WorkspaceViewContent 传递 kanbanActiveTaskId prop
  </action>
  <acceptance_criteria>
    - DndContext 只在 WorkspaceShell 中存在一个实例
    - KanbanView 内无 DndContext 相关 import
    - DragOverlay 在 WorkspaceShell 层渲染
    - TypeScript 编译通过
    - vite build 通过
  </acceptance_criteria>
</task>

# Plan 12-01: KanbanView 迁移到 @dnd-kit

## 目标

将 KanbanView 的手写 pointer events 拖拽替换为 @dnd-kit。本 Plan 先在 KanbanView 内嵌 DndContext，快速验证迁移可行性。下一 Plan 再将 DndContext 提升到 WorkspaceShell。

## Tasks

<task id="12-01-T1">
  <title>KanbanView — 用 @dnd-kit 替换手写 pointer events</title>
  <read_first>
    - web/src/components/views/KanbanView.tsx
    - web/src/components/views/KanbanView.module.css
    - web/src/components/shared.tsx (DragPreviewLayer)
  </read_first>
  <action>
    完全重写 KanbanView.tsx 拖拽逻辑：

    1. 删除所有 pointer events 状态/refs/useEffect
    2. 引入 @dnd-kit/core: DndContext, DragOverlay, useDraggable, useDroppable, MouseSensor, TouchSensor, useSensors
    3. DndContext 包裹整个 kanbanGrid，配置 sensors:
       - MouseSensor: activationConstraint { distance: 6 }
       - TouchSensor: activationConstraint { delay: 200, tolerance: 5 }
    4. 每个列（todo/doing/done）使用 useDroppable({ id: status })
    5. 每个任务卡片使用 useDraggable({ id: task.id })
    6. DragOverlay 渲染拖拽中的卡片预览（复用 .drag-preview-card 样式）
    7. onDragEnd: 检查 over?.id 是否为合法 TaskStatus，调用 onDropStatusChange
    8. 移除 DragPreviewLayer 组件的使用（不再需要）

    保留的 prop 接口不变：onDropStatusChange(taskId, status)

    KanbanDraggableCard 子组件：
    - 使用 useDraggable hook
    - isDragging 时应用 is-dragging class
    - 卡片内部 badge group 包一层 div + onPointerDown stopPropagation 防止触发 drag

    KanbanDroppableColumn 子组件：
    - 使用 useDroppable hook
    - isOver 时应用 is-drag-over class
  </action>
  <acceptance_criteria>
    - 删除所有 PointerDragSession / PointerDragPreviewState / DragPreviewLayer 引用
    - 删除 window.addEventListener(pointermove/pointerup/pointercancel)
    - DndContext + useSensors 正确配置
    - 桌面鼠标拖拽可触发 onDropStatusChange
    - TypeScript 严格模式编译通过（noUnusedLocals, noUnusedParameters）
    - vite build 通过
  </acceptance_criteria>
</task>

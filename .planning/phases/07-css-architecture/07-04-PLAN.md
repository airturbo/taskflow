# Plan D: Mobile CSS Modules + Final Cleanup

## Goal
Extract remaining mobile-specific CSS from index.css into `.module.css` files for the 7 mobile components. After this plan, index.css should be nearly empty (< 100 lines of truly cross-cutting rules or empty). This completes the ARCH-02 requirement.

## Context
Mobile CSS is the hardest part of this phase because:
1. Mobile CSS is scattered across index.css (not contiguous)
2. Heavy use of `.is-phone` parent selector requires `:global(.is-phone)` in CSS Modules
3. Some mobile CSS overlaps with desktop view CSS (already extracted in Plans B/C)

After Plans B and C, the remaining index.css content should be primarily:
- Mobile-only sections (L3136–L6921 minus what B/C already extracted)
- Any cross-cutting rules not yet extracted

# Plan B: Desktop Component CSS Modules Extraction

## Goal
Extract component-specific CSS from index.css into `.module.css` files for each desktop component. Update each TSX file to import its module and use `styles.className` syntax. Handle `.is-phone` references using `:global(.is-phone)`.

## Strategy
- Each component gets its own `.module.css` file
- CSS classes are converted from kebab-case to camelCase in TSX (e.g., `app-shell` → `styles.appShell`)
- `.is-phone` parent selectors use `:global(.is-phone) .localClass` pattern
- Responsive `@media` rules for a component go into that component's module
- After extraction, remove the corresponding sections from index.css
- **Section ownership**: Plan B owns index.css L126–L670, L815–L1160, L1161–L1422, L2363–L2736. Plan C owns L1423–L2362 (views). No overlap.

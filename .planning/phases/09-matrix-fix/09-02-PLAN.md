# Plan 09-02: 数据迁移与持久化更新

## 目标

在 `storage.ts` 的 `mergePersistedState` 中加入迁移逻辑：从旧数据的 tagIds 推断 isUrgent/isImportant，并清理特殊标签出 tagIds。同时更新 `useTaskActions.moveTaskToQuadrant` 改用独立字段。

## Tasks

<task id="09-02-T1">
  <title>storage.ts — mergePersistedState 加迁移逻辑</title>
  <read_first>
    - web/src/utils/storage.ts
  </read_first>
  <action>
    在 `mergePersistedState` 的 tasks.map 中，对每个 task 执行迁移：
    1. 检查是否有 isUrgent 字段（类型断言 `(task as unknown as {isUrgent?: boolean}).isUrgent`）
    2. 如果没有（旧数据），从 tagIds 推断
    3. 清理 tagIds 中的特殊标签（'tag-urgent', 'tag-important'）
    
    迁移逻辑（保持 task 类型安全）：
    ```typescript
    const rawTask = task as Task & { isUrgent?: boolean; isImportant?: boolean }
    const isUrgent = rawTask.isUrgent ?? rawTask.tagIds.includes('tag-urgent')
    const isImportant = rawTask.isImportant ?? rawTask.tagIds.includes('tag-important')
    const tagIds = task.tagIds.filter(id => id !== 'tag-urgent' && id !== 'tag-important')
    ```
  </action>
  <acceptance_criteria>
    - 旧数据加载后 isUrgent/isImportant 正确设置
    - tagIds 中不再含特殊标签
    - TypeScript 编译通过
  </acceptance_criteria>
</task>

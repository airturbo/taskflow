# Plan 09-01: 新增 isUrgent/isImportant 独立字段

## 目标

在 `Task` 接口新增 `isUrgent: boolean` 和 `isImportant: boolean` 字段，更新 `getQuadrant` 和 `getTagIdsForQuadrant` 逻辑，并在 seed 数据中填充初始值。

## Tasks

<task id="09-01-T1">
  <title>更新 Task 接口 — 新增 isUrgent/isImportant 字段</title>
  <read_first>
    - packages/taskflow-core/src/domain.ts
  </read_first>
  <action>
    在 Task 接口中新增两个布尔字段：
    ```typescript
    isUrgent: boolean
    isImportant: boolean
    ```
    放在 tagIds 字段之后。
  </action>
  <acceptance_criteria>
    - Task 接口有 isUrgent 和 isImportant 布尔字段
    - TypeScript 严格模式通过
  </acceptance_criteria>
</task>

<task id="09-01-T2">
  <title>更新 getQuadrant — 改读独立字段</title>
  <read_first>
    - packages/taskflow-core/src/selectors.ts
  </read_first>
  <action>
    修改 `getQuadrant(task)` 使用 `task.isUrgent` 和 `task.isImportant`。
    新增 `getFieldsForQuadrant(quadrant)` 替代 `getTagIdsForQuadrant`，返回 `{ isUrgent: boolean, isImportant: boolean }`。
    保留 `getTagIdsForQuadrant` 但只做 tagIds 清理（移除特殊标签），不再添加特殊标签。
  </action>
  <acceptance_criteria>
    - getQuadrant 读 isUrgent/isImportant 而非 tagIds
    - getFieldsForQuadrant 正确返回每个象限的字段值
    - TypeScript 编译通过
  </acceptance_criteria>
</task>

<task id="09-01-T3">
  <title>更新 seed 数据 — 填充 isUrgent/isImportant</title>
  <read_first>
    - web/src/data/seed.ts
  </read_first>
  <action>
    为 seed.ts 中所有 task 添加 isUrgent/isImportant 字段，根据已有 tagIds 推断：
    - 包含 'tag-urgent' → isUrgent: true
    - 包含 'tag-important' → isImportant: true
    - 其他 → false
    同时从 tagIds 中移除 'tag-urgent' 和 'tag-important'。
  </action>
  <acceptance_criteria>
    - 所有 seed task 都有 isUrgent/isImportant 字段
    - seed task 的 tagIds 不再包含 'tag-urgent'/'tag-important'
    - TypeScript 编译通过
  </acceptance_criteria>
</task>

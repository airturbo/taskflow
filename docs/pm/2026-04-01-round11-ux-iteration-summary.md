# 2026-04-01 产品团队第 11 轮迭代总结

## 本轮目标

在上一轮修完看板拖拽、时间线拖动、状态显著化和提醒/番茄基础反馈之后，继续把几个“已经能用但还不够成熟”的环节往真实产品再推一步：

1. 给时间线补上真正可见、可切换的日/周尺度与独立导航。
2. 让提醒中心从“展示最近提醒”升级为“提醒后的恢复台”，支持稍后提醒。
3. 给看板拖拽补上更轻但明确的完成反馈，并允许立即撤销。
4. 修正番茄倒计时在页面刷新/恢复场景下的可靠性问题。

---

## 本轮落地

### 1. 时间线升级为日 / 周双尺度
- 新增时间线工具条，支持上一窗口 / 今天 / 下一窗口 / 日期选择器。
- 新增“日 / 周”切换：
  - **周** 视图继续承担整周统筹。
  - **日** 视图聚焦当天排班，拖动时采用更细的 15 分钟吸附。
- 创建入口也随尺度变化：
  - 周视图保留按天快速创建。
  - 日视图提供 09:00 / 13:00 / 18:00 / 21:00 四个快速排期槽。

### 2. 提醒中心支持稍后提醒
- 触发后的任务提醒与到期提醒，现在会携带任务上下文进入提醒中心。
- 每条可恢复提醒都提供：
  - `10 分钟后`
  - `1 小时后`
- 点击后会为原任务追加一条绝对提醒，并写入任务 activity，提醒中心也会给出成功反馈。
- 默认权限提示文案也做了收口：未授权时明确说明“仍会保留应用内提醒与提示音”。

### 3. 看板拖拽补上微反馈与撤销
- 卡片跨列拖拽落下后，除了状态立即变化外，还会出现一条轻量 toast：
  - 告知“已移到某状态”
  - 显示任务标题
  - 提供立即撤销
- 这条反馈是刻意做轻的，不抢视线，但能消除“到底改没改成功”的不确定感。

### 4. 番茄计时可靠性修正
- 修复了专注倒计时剩余秒数的计算问题，避免时间基准错误导致计时失真。
- 本地持久化恢复时，不再一律把运行中的番茄清空；如果刷新时回合尚未结束，会按剩余时长恢复。
- 这一步不是新 feature，但对“真能用”很关键。

---

## 当前判断

这轮不是单纯再堆一个 feature，而是继续把几个关键链路从“功能存在”推进到“使用时更顺、更稳、更像成熟工具”：

- 时间线现在既能统筹整周，也能细排当天。
- 提醒触发后不再只能被动看到，而能直接恢复到下一次提醒。
- 看板拖拽终于有了短促但完整的确认闭环。
- 番茄计时的连续性不再脆弱到刷新就丢。

## 下一轮建议

如果继续沿这个标准推进，下一轮优先级建议是：

1. 把提醒中心做成可清理 / 可筛选的短历史，而不只是最近 6 条。
2. 给时间线补 pointer 事件，解决触屏与平板拖动体验。
3. 让番茄白噪音从“选择态”变成真正可播放的音频能力，或者明确降级隐藏。
4. 把看板拖拽反馈扩展到时间线调整后的微反馈，形成一致的工作台语言。

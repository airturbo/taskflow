## 一、需要立即执行的改动（用户明确提出）

### 1. 全局：去掉所有 Tab 顶部的快速创建栏

**现状**：每个 Tab 顶部都有"例如：明天下午3点产品评审 + 立即创建"的输入栏。  
**问题**：右下角已有 FAB 创建按钮，顶部栏重复且浪费空间。  
**改法**：手机端（`isPhoneViewport`）隐藏 `.composer-bar` 区域。

```css
.is-phone .composer-bar { display: none; }
```

### 2. 焦点页：去掉批量操作

**现状**：焦点页顶部有"批量操作"按钮，但只能全选，无实际意义。  
**改法**：手机端焦点页隐藏批量操作栏。

```css
.is-phone .bulk-toolbar { display: none; }
```

### 3. 焦点页：勾选框缩小

**现状**：勾选圆太大（约 32-36px），占据过多卡片空间。  
**改法**：手机端勾选按钮进一步缩至 `16px`，border `1.5px`。

```css
.is-phone .check-button,
.is-phone .mobile-focus-card__check {
  width: 16px; height: 16px; min-width: 16px; min-height: 16px;
  border-width: 1.5px; flex-shrink: 0;
}
```

### 4. 焦点页：顶栏 "TaskFlow" 改为 scope 切换入口

**现状**：顶栏左上角显示 "TaskFlow"，浪费位置。  
**改法**：把焦点页的 scope 下拉按钮（全部/今日/未来7天/清单）移到顶栏 "TaskFlow" 的位置。顶栏不再显示 "TaskFlow" 文字，而是显示当前 scope 名称 + ▾。

```
改前：[≡] TaskFlow          [🌙] [⏰] [登录]
改后：[≡] 全部 ▾             [🌙] [⏰] [登录]
       ↑ 点击展开 scope 菜单
```

当选择"今日"或"未来7天"时，在 scope 按钮右侧紧跟一个小的"按计划 | 按DDL"切换 pill。

### 5. 日历页：去掉顶部快捷创建（同第1点）

### 6. 日历页：周历模式去掉重复的星期头

**现状**：周历上方有一行"日 一 二 三 四 五 六"，每天格子下方又有一行"日 一 二 三…"，导致视觉重复浪费空间。  
**改法**：手机端周历只保留一行星期标头，每天显示日期+农历即可，每天是一个紧凑的纵向列。参考滴答清单截图的周视图：顶部一行日期，选中日高亮，下方显示选中日任务。

### 7. 日历页：视图切换改到顶栏左上角

**现状**：月历/周历/日历切换在页面底部区域。  
**改法**：顶栏左上角改为一个视图切换入口（类似滴答清单：点击出下拉菜单），选项为"月历 / 周历 / 日历"。顶栏中间保留月份标题。

       ↑ 点击弹出：月历/周历/日历

### 8. 日历页：周历和月历的正确交互模式

参考滴答清单截图：

**月历模式**：
- 上半部分是紧凑的月历网格（每格只有日期+农历+小圆点标注有任务的日期）
- 点击某一天，下半部分显示当天的任务列表
- 月历格子高度尽量压缩（36-40px），留更多空间给任务列表

**周历模式**：
- 上半部分是 7 天横向排列（仅一行，显示日期和星期）
- 选中日高亮
- 下半部分显示选中日任务列表

**日历/列表模式**：
- 按日期分组的任务列表（agenda 模式）

### 9. 象限 Tab：卡片体积压缩一半以上

**现状**：四象限内每个卡片体积太大，一屏看不完四象限。  
**改法**：
- 象限卡片高度压缩至 28-32px，只显示标题（单行截断）+ 可选的时间标签
- 勾选圆 14px
- padding `4px 8px`
- font-size `12px`
- 点击卡片 → 弹出摘要弹窗（标题+时间+优先级+清单），弹窗底部"展开详情"按钮打开完整 Sheet

### 10. 象限 Tab：看板视图改为一屏三列

**现状**：看板是单列切换模式。  
**改法**：手机端看板改为一屏三列（待办/进行中/完成），每列宽度 1/3。卡片极紧凑（同象限卡片规格），只显示标题+时间。点击卡片弹摘要弹窗再展开详情。

### 11. 象限 Tab：时间线视图点击弹详情，长按拖动

**现状**：时间线的交互不够明确。  
**改法**：
- 点击时间轴上的任务条 → 弹出详情 Sheet
- 长按拖动 → 调整时间轴起止点
- 普通触摸不触发任何拖拽

### 12. 取消"项目" Tab

**现状**：底部 5 个 Tab（焦点/日历/象限/项目/我的）。  
**改法**：去掉"项目"Tab，改为 4 个 Tab（焦点/日历/象限/我的）。

项目/清单的访问方式改为：
- 焦点页的 scope 下拉菜单中可切到任意清单
- "我的"页面中保留清单/文件夹管理入口

### 13. "我的"页：完善标签管理

**现状**：标签管理交互简陋。  
**改法**：参考主流 App 的标签管理：
- 标签列表可编辑（名称、颜色）
- 支持新建、删除、排序
- 每个标签显示关联任务数量
- 用 Sheet 形式呈现编辑界面

### 14. 名称问题

确保所有出现 "Todo Workspace" 或 "工作台" 的地方都已替换为 "TaskFlow"（上一轮应已完成，需验证）。

## 二、体验官自测发现的问题（代码审查 + 模拟操作）

> 以下是我逐页审查代码和模拟手机操作后发现的问题，用户未提但实际影响体验。

### E1. 焦点页 scope 菜单缺少点击外部关闭

**现状**：`focusScopeMenuOpen` 只在菜单项点击时关闭，**点击菜单外区域不会关闭**。  
**影响**：用户展开菜单后不想选择、点背景，菜单不收回。  
**改法**：添加 `useEffect` 监听 document click 事件，菜单外点击时关闭。

### E2. 焦点页 scope 菜单没有遮罩/背景

**现状**：菜单是 `position: absolute` 弹出但没有半透明遮罩。  
**影响**：菜单弹出时下方内容仍可点击，容易误触。  
**改法**：菜单弹出时加一层透明遮罩层拦截点击事件。

### E3. MobileFocusCard 没有触摸反馈

**现状**：点击卡片没有任何视觉反馈（无 active 态、无 ripple）。  
**影响**：用户不确定是否点到了，手感"木"。  
**改法**：添加 `:active` 态 `opacity: 0.7` + `transform: scale(0.98)` + transition。

### E4. 手机端顶栏 `[≡]` 汉堡菜单仍然可见但多余

**现状**：`isNavigationDrawerMode` 时显示汉堡菜单按钮，点击打开左侧导航抽屉。但手机端已有底部 Tab 导航，左侧抽屉内容与底部 Tab 重复。  
**影响**：两套导航系统并存，用户困惑"侧栏和底栏有什么区别"。  
**改法**：手机端隐藏顶栏汉堡菜单按钮（保留非手机平板端的行为）。

### E5. 我的页 `MobileMeView` 缺少清单管理入口

**现状**：只有标签管理、已完成、回收站入口。如果取消"项目"Tab，**用户完全无法进入清单管理和文件夹管理**。  
**影响**：取消项目 Tab 后的功能缺失。  
**改法**："我的"页增加"清单管理"行，点击跳到清单列表页（复用 `MobileProjectsView`）。

### E6. 快速创建 Sheet 输入框 font-size < 16px

**现状**：`MobileQuickCreateSheet` 的 `.mobile-quick-create-input` 未显式设置 font-size 16px。  
**影响**：iOS Safari 自动放大页面。  
**改法**：CSS 强制 `font-size: 16px`。

### E7. 所有 Tab 切换没有过渡动画

**现状**：底部 Tab 切换时内容区直接替换，没有淡入淡出或滑动过渡。  
**影响**：切换感觉生硬、不精致。  
**改法**：内容区加 `opacity 0.15s ease` 过渡，切换时短暂淡出再淡入。

### E8. FAB 按钮在象限页会遮挡右下角象限内容

**现状**：FAB 固定在右下角 `bottom: 76px`，象限 2x2 布局时右下象限内容被遮挡。  
**影响**：无法看到或点击右下象限底部的任务。  
**改法**：象限页 FAB 改为更小尺寸或半透明态，或者象限页底部留出 FAB 高度的 padding。

### E9. 详情 Sheet 内的提醒区域过于复杂

**现状**：提醒编辑器（`ReminderComposer`）有多个锚点选择、预设按钮、自定义输入，在手机 Sheet 中堆叠过长。  
**影响**：Sheet 内容区纵向太长，但因为滚动问题（用户反馈的第 5 点），很多内容看不到。  
**改法**：手机端提醒区域默认折叠，只显示"已设 n 条提醒 + 添加"，点击展开完整编辑器。

### E10. 手机端搜索入口缺失

**现状**：桌面端有搜索框，但手机端没有任何搜索入口。  
**影响**：任务多时完全无法搜索。  
**改法**：顶栏增加搜索图标按钮，点击展开搜索框（覆盖顶栏区域），输入时实时过滤任务。

### E11. 任务完成后无视觉反馈

**现状**：点击勾选按钮完成任务后，卡片立即从列表消失（如果过滤条件排除已完成）。  
**影响**：用户不确定"到底完成了没有"，体验突兀。  
**改法**：完成后卡片先播放 `opacity → 0 + translateX(20px)` 的淡出动画（200ms），然后才从列表移除。同时底部弹出 "已完成 · 撤销" 的 Toast（3 秒自动消失）。

### E12. 手机端 window.prompt / window.confirm 残留

**现状**：虽然已新增 `MobileConfirmSheet` 和 `MobilePromptSheet`，但 `App.tsx` 中清单管理部分（新建、重命名、删除）仍直接调用 `window.prompt()` 和 `window.confirm()`（行 ~1920-2000 的 navigationContent 区域，行 ~2085-2095 的 ctxMenu 区域）。  
**影响**：手机端弹出浏览器原生弹窗，极其丑陋且不匹配产品气质。  
**改法**：把这些 `window.prompt/confirm` 调用逐一替换为 `mobilePrompt/mobileConfirm`。

### E13. 视图切换不保持滚动位置

**现状**：从焦点切到日历再切回来，焦点页的滚动位置丢失（回到顶部）。  
**影响**：频繁切 Tab 的用户体验割裂。  
**改法（后续优化）**：用 `useRef` 记录每个 Tab 的 scrollTop，切换时恢复。

## 三、外部审查文档值得采纳的建议

> 来源：`/Users/turbo/WorkBuddy/20260330162606/移动端UX优化落地方案.md`  
> 筛选标准：与用户反馈不冲突 + 实际有体验价值 + 当前可落地

### D1. iOS 输入框防缩放（文档 Task 1.5）

**采纳**。手机端所有 `input`/`textarea`/`select` 的 font-size 强制 `16px`。

### D2. 任务卡片优先级左边框色条（文档 Task 2.5 附带）

**采纳**。用 3px 左边框色条标识优先级（红=紧急、橙=高、蓝=中、灰=低），比右上角标签 chip 更不占空间且视觉更优雅。

### D3. 详情 Sheet 三层信息架构（文档 Task 2.3）

**采纳**。把当前平铺的详情字段改为：
- 第一层（始终可见）：标题、时间、优先级、清单
- 第二层（默认展开）：备注、标签、提醒摘要
- 第三层（折叠 `<details>`）：开始时间、DDL、子任务、附件、协作者

配合底部固定操作栏（完成 / 删除）。

### D4. 空状态情感化（文档 Task 3.5）

**采纳**。分场景设计空态：
- 焦点页全部完成："🎉 今天的任务都完成了！"
- 焦点页无任务："☀️ 今天还没有安排"
- 清单为空："📋 添加第一个任务"
- 搜索无结果："🔍 没有找到相关任务"

### D5. 任务完成后的撤销 Toast（文档 Task 3.3）

**采纳**。完成/删除操作后显示底部 Toast：`"已完成" [撤销]`，3 秒后自动消失。这是 Things 3 / Todoist 的标配交互。

### D6. 三段时间模型渐进披露（文档末尾专节）

**采纳方向**。新用户默认只看到"截止时间"（dueAt），高级用户可在设置中开启"专业模式"显示 startAt + deadlineAt。但当前优先级低，作为后续迭代。

### D7. 看板单列模式（文档 Task 2.6）

**部分采纳**。外部文档建议完全单列，用户要求一屏三列。**以用户要求为准**：手机端看板保持一屏三列（紧凑卡片），但同时提供单列切换选项供用户自行选择。

### D8. 统一断点 useDevice hook（文档 Task 1.3）

**已落地**。之前的重构已经统一了 680px 断点，该建议已实现。

### D9. 视图滚动位置保持（文档 Task 3.4）

**采纳方向**，对应体验官发现的 E13。后续迭代时实现。

## 七、交互层级统一

所有视图中点击任务卡片的行为统一为：

1. **点击** → 弹出**摘要弹窗**（标题 + 时间 + 优先级 + 清单 + "展开详情"按钮）
2. **展开详情** → 全屏任务详情 Sheet（三层信息架构，可编辑）
3. **长按**（仅在支持拖拽的视图中）→ 进入拖拽模式

这样避免"点击直接弹出完整详情 Sheet 太重"的问题，同时保留深度操作路径。

# TaskFlow 手机端 UX / UI 重构建议

> **设计目标**：将桌面端的多视图任务管理能力，转化为符合移动端操作习惯的流畅体验  
> **参考对标**：Notion Mobile、滴答清单、Things 3  
> **核心原则**：不抄具体样式，吸收交互智慧

---

## 1. 核心设计原则

### 1.1 拇指优先（Thumb-First）
- **关键操作区域**：屏幕下方 1/3 为主要交互区，避免顶部密集按钮
- **触摸目标**：最小 44×44pt（iOS HIG）/ 48×48dp（Material Design）
- **手势优先级**：
  - 一级操作：单击、长按、底部按钮
  - 二级操作：左右滑动、下拉刷新
  - 三级操作：双指捏合、多点触控

### 1.2 渐进式信息披露（Progressive Disclosure）
- **默认视图**：只展示核心信息（标题、时间、状态）
- **展开详情**：上滑 Sheet / 点击卡片进入全屏详情
- **减少层级**：任何操作不超过 3 步

### 1.3 上下文感知（Context-Aware）
- **智能默认**：根据当前时间/位置/历史行为预填字段
- **快速切换**：视图间切换保留滚动位置和筛选状态
- **动态适配**：横屏自动切换为 Compact 布局

### 1.4 流畅反馈（Fluid Feedback）
- **即时响应**：触摸反馈 < 100ms
- **有意义的动效**：过渡动画传达空间关系（如 Sheet 从底部滑出 = 叠加层级）
- **离线优先**：操作立即生效，后台同步

### 1.5 减少认知负担
- **一屏一焦点**：避免同时展示日历 + 列表 + 看板
- **渐进式引导**：首次使用时通过空态引导核心操作
- **容错设计**：支持撤销、草稿自动保存

## 2. 信息架构重构

### 2.1 当前桌面端架构问题
```
桌面端：侧边栏（清单） + 顶栏（视图切换） + 主区域（内容） + 右侧栏（详情/统计）
问题：
- 移动端无法同时展示 4 个区域
- 视图切换器占用顶部黄金位置
- 详情抽屉与主内容冲突
```

### 2.2 移动端三层架构

#### **Layer 1: 导航层（底部 Tab Bar）**
```
┌─────────────────────────────────────┐
│                                     │
│         主内容区域                   │
│                                     │
└─────────────────────────────────────┘
┌─────┬─────┬─────┬─────┬─────┐
│ 今天 │ 收件箱│ 清单 │ 搜索 │ 我的 │  ← 固定 5 个入口
└─────┴─────┴─────┴─────┴─────┘
```

**设计依据**：
- **今天**：最高频入口（对标 Things 3 "Today"）
- **收件箱**：快速捕获（对标滴答清单 "Inbox"）
- **清单**：项目视图（对标 Notion "Workspace"）
- **搜索**：全局检索
- **我的**：设置、统计、账户

#### **Layer 2: 视图层（顶部 Segmented Control）**
```
进入"今天"或"清单"后，顶部出现：
┌──────────────────────────────────┐
│  列表  |  日历  |  看板  |  矩阵  │  ← 横向滚动
└──────────────────────────────────┘
```

**设计依据**：
- 仅在需要时出现（避免全局污染）
- 支持左右滑动切换视图（手势 + 按钮双通道）
- 当前视图高亮 + 下划线指示器

#### **Layer 3: 操作层（浮动 FAB + Sheet）**
```
主内容区右下角：
    ┌───┐
    │ + │  ← 浮动按钮（FAB）
    └───┘
点击后弹出 Bottom Sheet：
┌─────────────────────────────────────┐
│  快速添加任务                        │
│  ┌─────────────────────────────┐   │
│  │ 输入框（支持智能识别）        │   │
│  └─────────────────────────────┘   │
│  [ 今天 ] [ 明天 ] [ 下周 ]         │
│  [ 重要 ] [ 紧急 ]                  │
└─────────────────────────────────────┘
```

## 3. 核心场景移动端交互方案

### 3.1 日历视图（Calendar）

#### 当前桌面端问题
- 月视图在手机上格子太小，无法点击
- 周视图信息密度过高
- Agenda 列表与日历分离

#### 移动端方案：**混合式日历**
```
┌─────────────────────────────────────┐
│  2026年3月              [今天]       │  ← 顶部导航
├─────────────────────────────────────┤
│  日 一 二 三 四 五 六                │
│  23 24 25 26 27 28 29               │  ← 迷你月历（可左右滑动切换月份）
│      ●     ●  ●                     │  ← 圆点标记有任务的日期
├─────────────────────────────────────┤
│  今天 · 3月27日 周四                │  ← 选中日期标题
│                                     │
│  ┌─ 09:00 ─────────────────────┐  │
│  │ 团队周会                     │  │  ← 时间轴任务卡片
│  │ 会议室 A                     │  │
│  └──────────────────────────────┘  │
│                                     │
│  ┌─ 14:00 ─────────────────────┐  │
│  │ 产品评审                     │  │
│  └──────────────────────────────┘  │
│                                     │
│  ── 未定时间 ──                    │
│  □ 回复客户邮件                    │  ← 当日无时间任务
│  □ 整理会议纪要                    │
└─────────────────────────────────────┘
```

**交互细节**：
1. **日期选择**：点击迷你日历日期 → 下方列表自动滚动到对应日期
2. **快速跳转**：顶部"今天"按钮 → 回到当前日期
3. **左右滑动**：在任务列表区域左右滑动 → 切换前后日期
4. **长按日期**：弹出快速创建菜单（"在此日期创建任务"）
5. **下拉刷新**：同步最新数据

**视觉优化**：
- 迷你日历高度固定 80pt，避免占用过多空间
- 任务卡片左侧色条标识优先级
- 时间轴采用相对定位（避免绝对时间刻度）

### 3.2 列表视图（List）

#### 当前桌面端问题
- 多列信息在手机上换行混乱
- 批量操作依赖复选框（触摸不友好）

#### 移动端方案：**卡片式列表 + 滑动操作**
```
┌─────────────────────────────────────┐
│  收件箱                   [筛选]     │
├─────────────────────────────────────┤
│  ┌───────────────────────────────┐ │
│  │ □ 完成季度报告               │ │  ← 任务卡片
│  │   📌 重要 · DDL 3/30         │ │
│  │   #工作 #报告                │ │
│  └───────────────────────────────┘ │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ □ 预约体检                   │ │
│  │   明天 10:00                 │ │
│  └───────────────────────────────┘ │
└─────────────────────────────────────┘
```

**滑动操作**（参考 Things 3）：
```
向右滑动 →  [ ✓ 完成 ]
向左滑动 ←  [ 🗓️ 改期 ] [ 🗑️ 删除 ]
长按      →  进入多选模式
```

**多选模式**：
- 长按任意任务 → 左侧出现圆形复选框 + 底部工具栏
- 底部工具栏：[ 完成 ] [ 移动 ] [ 标签 ] [ 删除 ]
- 顶部显示"已选 3 项" + 取消按钮

**分组逻辑**：
- 默认按日期分组（逾期 / 今天 / 明天 / 本周 / 未来）
- 支持切换为按清单 / 按标签 / 按优先级分组

### 3.3 看板视图（Kanban）

#### 当前桌面端问题
- 三列横向布局在手机上需要频繁横向滚动
- 拖拽卡片在触摸屏上容易误触

#### 移动端方案：**纵向堆叠 + 快速切换**
```
┌─────────────────────────────────────┐
│  看板                               │
│  ┌─────┬─────┬─────┐               │
│  │ 待办 │ 进行中│ 完成 │  ← 顶部 Tab │
│  └─────┴─────┴─────┘               │
├─────────────────────────────────────┤
│  待办 (12)                          │  ← 当前列标题
│                                     │
│  ┌───────────────────────────────┐ │
│  │ 设计移动端原型               │ │
│  │ #设计 · DDL 3/28             │ │
│  └───────────────────────────────┘ │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ 编写技术文档                 │ │
│  │ #开发                        │ │
│  └───────────────────────────────┘ │
└─────────────────────────────────────┘
```

**交互方案**：
1. **切换列**：点击顶部 Tab / 左右滑动卡片区域
2. **移动卡片**：
   - 长按卡片 → 进入拖拽模式（卡片放大 + 震动反馈）
   - 拖到屏幕边缘 → 自动切换到相邻列
   - 松手 → 卡片落入新列（弹性动画）
3. **快速操作**：
   - 向右滑动卡片 → [ → 进行中 ]
   - 向左滑动卡片 → [ ✓ 完成 ]

**替代方案**（降低拖拽依赖）：
- 点击卡片进入详情 → 底部"移动到"按钮 → 选择目标列

### 3.8 拖拽与排序

#### 当前桌面端问题
- 鼠标拖拽在触摸屏上容易误触
- 拖拽范围受限于视口

#### 移动端方案：**长按触发 + 边缘自动滚动**
```
长按任务卡片 → 进入拖拽模式：
┌─────────────────────────────────────┐
│  ↑ (拖到这里向上滚动)                │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ 📌 正在拖拽的任务             │ │  ← 卡片放大 + 阴影
│  └───────────────────────────────┘ │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ 其他任务 (可插入这里)         │ │  ← 插入位置指示线
│  └───────────────────────────────┘ │
│                                     │

```

**交互细节**：

4. **跨视图拖拽**：

## 4. 视觉与动效方向

### 4.1 视觉风格建议

#### **方向 1：现代极简（推荐）**
- **色彩**：大面积留白 + 单一强调色（可自定义）
- **字体**：SF Pro / Roboto，标题 18-20pt，正文 15-16pt
- **圆角**：统一 12pt（卡片）、8pt（按钮）
- **阴影**：轻微投影（0 2px 8px rgba(0,0,0,0.08)）
- **参考**：Things 3、Apple Reminders

#### **方向 2：卡片层叠**
- **色彩**：浅色背景 + 彩色卡片
- **间距**：卡片间距 12pt，内边距 16pt
- **动效**：卡片堆叠时显示轻微错位（3D 效果）
- **参考**：Notion Mobile、Trello

#### **方向 3：时间线可视化**
- **色彩**：时间轴用渐变色标识时段
- **图形**：任务卡片左侧色条 + 时间刻度
- **动效**：时间流逝动画（进度条）
- **参考**：Google Calendar、Fantastical

### 4.2 动效设计原则

#### **有意义的动画**（避免无意义的炫技）
```
✓ 好的动效：
- Sheet 从底部滑出 → 传达"叠加层级"
- 任务完成后淡出 → 传达"消失"
- 拖拽时卡片放大 → 传达"正在操作"

✗ 差的动效：
- 页面切换旋转 360° → 无意义
- 按钮点击爆炸特效 → 干扰注意力
```

#### **动画时长**
- **微交互**：100-200ms（按钮点击、复选框勾选）
- **页面切换**：300-400ms（视图切换、Sheet 滑出）
- **拖拽反馈**：实时跟随（0 延迟）

#### **缓动函数**
- **进入**：ease-out（快速启动，平滑结束）
- **退出**：ease-in（平滑启动，快速结束）
- **交互**：spring（弹性动画，更自然）

#### **关键动效场景**
1. **任务完成**：
   - 复选框 → 勾选动画（弹性）

## 6. 技术实现建议

### 6.1 响应式断点策略
```css
/* 当前断点过于复杂，建议简化为 3 档 */
@media (max-width: 480px)  { /* 手机竖屏 */ }
@media (max-width: 768px)  { /* 手机横屏 / 小平板 */ }
@media (max-width: 1024px) { /* 平板 */ }
```

### 6.2 触摸手势库
推荐使用 `react-use-gesture` 或 `framer-motion` 的手势支持：
```tsx
import { useGesture } from '@use-gesture/react'

const bind = useGesture({
  onDrag: ({ movement: [mx], direction: [dx], cancel }) => {
    if (Math.abs(mx) > 100) {
      if (dx > 0) handleComplete()
      else handleDelete()
      cancel()
    }
  }
})
```

### 6.3 虚拟滚动优化
对于长列表（> 100 项），使用 `react-window` 或 `@tanstack/react-virtual`：
```tsx
import { useVirtualizer } from '@tanstack/react-virtual'

const rowVirtualizer = useVirtualizer({
  count: tasks.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 72, // 任务卡片高度
})
```

### 6.4 动画性能优化
```css
/* 只对 transform 和 opacity 做动画（GPU 加速）*/
.task-card {
  transition: transform 0.3s ease-out, opacity 0.3s ease-out;
  will-change: transform, opacity; /* 提前告知浏览器 */
}

/* 避免对 width/height/top/left 做动画（触发重排）*/
```

## 8. 风险与注意事项

### 8.1 技术风险
- **性能问题**：大量任务（> 500）时列表滚动卡顿
  - **缓解**：虚拟滚动 + 分页加载
- **拖拽冲突**：拖拽与滚动手势冲突
  - **缓解**：长按延迟 500ms + 拖拽时禁用滚动

### 8.2 体验风险
- **学习成本**：桌面端用户迁移到移动端需要重新学习
  - **缓解**：首次使用引导 + 手势提示
- **功能缺失**：移动端无法实现所有桌面功能（如批量导入）
  - **缓解**：在"我的"页面提示"部分高级功能请在桌面端使用"

### 8.3 兼容性风险
- **iOS vs Android**：手势习惯不同（如返回手势）
  - **缓解**：遵循各平台设计规范（iOS HIG / Material Design）
- **旧设备**：iPhone 8 / Android 8 性能较差
  - **缓解**：降级动画（检测设备性能）
